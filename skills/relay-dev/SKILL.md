# Relay Dev

## 目标
作为接力开发的统一入口，让用户只用中文自然语言就能触发正确的接力流程，不需要记英文 skill 名。

## 中文触发话术
- `接手这个项目，按接力开发规则来`
- `给这个项目初始化接力开发`
- `重新同步当前任务`
- `做一次标准交接`
- `按接力开发流程处理这个项目`

## 分流规则
1. 如果用户在新项目上要求铺模板、初始化规则文件、生成交接文件，进入“初始化模式”：
   - 提醒使用接力开发母包初始化命令。
   - 目标是生成 `AGENTS.md`、`progress.md`、`task_plan.md`、`findings.md`、`task_registry.md`、`修改记录_会话备忘.md`、提示词文件与桥接规则。
2. 如果用户是新会话接手或要求“按接力开发规则接手”，转到 `relay-start`：
   - 按顺序恢复上下文。
   - 锁定 `Task-ID`。
   - 输出 `Session Context` 并执行启动检查。
3. 如果用户说上下文乱了、漂移了、重新同步，转到 `relay-resync`：
   - 重新读取关键上下文文件。
   - 重新陈述 Active Goal / Current Phase / Next First Command。
4. 如果用户准备结束、切换工具或切换模型，转到 `relay-handoff-stop`：
   - 更新 `progress.md` 顶部快照。
   - 追加 `修改记录_会话备忘.md`。
   - 回写 `task_registry.md`。
5. 如果用户明确说“按接力开发流程处理这个项目”，先根据当前阶段判断：
   - 没有模板：按初始化模式。
   - 有模板但无上下文：按 `relay-start`。
   - 有模板但上下文冲突：按 `relay-resync`。
   - 正在收尾：按 `relay-handoff-stop`。

## 默认原则
1. 优先用中文与用户互动。
2. 默认采用“摘要优先、按需深读”恢复上下文，不机械全文通读长文件。
3. 先锁定单一 `Task-ID`，再继续执行。
4. 未验证前不宣称“已完成/已修复”。
