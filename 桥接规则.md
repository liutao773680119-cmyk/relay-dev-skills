# 桥接规则

## 适用场景
- 切换模型
- 切换工具
- 会话中断后恢复

## 最小桥接要求
- 先读交接文件，再做任何分析或修改。
- 先锁定当前 `Task-ID`，再继续执行。
- 如果上下文冲突，优先更新 `progress.md` 与 `修改记录_会话备忘.md`，再重述当前状态。
- 如果任务范围变化，必须先更新 `task_plan.md` 与 `task_registry.md`。
- 如果准备暂停，必须先更新 `Latest Handoff Snapshot` 和 `Session Record`。
- 如果本轮确认了错误路径、无效命令或无效假设，必须同步更新 `避坑记录.md`。
